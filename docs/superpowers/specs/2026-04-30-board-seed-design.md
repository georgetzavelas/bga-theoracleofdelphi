# Board Seed — Design

**Date:** 2026-04-30
**Status:** Design approved, ready for implementation plan
**Owner:** G

## Problem

Boards generated by `BoardGenerator` are unrepeatable. Once a game ends, the layout is gone — there's no way to:
1. Reproduce a specific board for debugging (e.g., "the board that broke pathfinding").
2. Let players know which board they played, share it, or play the same one again.
3. Record the board identity in BGA's stats system.

The board generator already accepts a `randFn` constructor option for testability ([BoardGenerator.php:32](../../modules/php/BoardGenerator.php)) but currently uses BGA's framework `bga_rand`, which is deterministic per *table seed* — not per *board*. There's no extracted, displayable board identifier.

## Goal

Capture a small, versioned identifier for every generated board, persist it across the table's lifetime, expose it via BGA's stats system, and provide a developer path to reproduce the board from the identifier.

**Non-goals:**
- Rendering the seed during live gameplay (no in-game UI). BGA's stats panel covers post-game visibility.
- JS-side board reproduction. PHP is the authoritative source.
- Keeping old seeds replayable across algorithm changes. Versioning explicitly walls this off — old seeds remain *recorded* but cannot be regenerated against newer code.

## Approach

Three concrete pieces:

1. **Seedable PRNG.** A small instance-scoped `SeededRandom` class implementing Mulberry32 in PHP. Replaces the default `bga_rand`-based randFn for the duration of one `BoardGenerator::generate()` call.
2. **Versioned seed identity.** A 32-bit integer seed paired with an integer algorithm version. Stored via BGA's globals mechanism. Reported via stats. Encoded for human-friendly display via Crockford base32 as `v1-K7F3-9DR`.
3. **PHP regeneration script.** A dev-only CLI that takes a decimal or encoded seed, validates the version, reproduces the board, and prints the placements.

Aspect-ratio bias from the prior change (target 1.5:1) remains in effect; the seed determines all bias-influenced randomness.

## Architecture

### New files
- `modules/php/SeededRandom.php` (~30 lines) — Mulberry32 PRNG class with one method: `rand(int $min, int $max): int`.
- `modules/php/BoardSeed.php` (~40 lines) — Static encode/decode between 32-bit ints and `v{version}-XXXX-XXX` Crockford base32 strings.
- `tests/regenerate_board.php` (~50 lines) — CLI that reproduces a board from a seed string or decimal. Accepts both encoded and decimal forms; rejects version mismatches.

### Modified files
- `modules/php/BoardGenerator.php` — Add `public const ALGORITHM_VERSION = 1;`. No other changes (the existing `randFn` slot is what we plug into).
- `modules/php/Game.php` — In `__construct`, register two `initGameStateLabels` entries (`board_seed_decimal => 20`, `board_algorithm_version => 21`). In `setupNewGame` around [line 2124](../../modules/php/Game.php), generate the seed via `bga_rand`, construct `SeededRandom` and pass it to `BoardGenerator`, then persist the seed and version via `setGameStateValue` and report them via `statInc`.
- `stats.json` — Add `board_seed_decimal` (id 11) and `board_algorithm_version` (id 12) under `table`.
- `tests/test_board_generator.php` — Add a determinism test (same seed → byte-identical board) and a non-determinism test (different seeds → different boards).

### Untouched
- `modules/js/BoardBuilder.js` — JS preview does not need to reproduce boards.
- `BoardRenderer.js`, the TPL, the CSS — no in-game UI.
- All other PHP modules.

## PRNG

```php
class SeededRandom {
    private int $state;

    public function __construct(int $seed) {
        $this->state = $seed & 0xFFFFFFFF;
    }

    public function rand(int $min, int $max): int {
        $this->state = ($this->state + 0x6D2B79F5) & 0xFFFFFFFF;
        $t = $this->state;
        $t = (($t ^ ($t >> 15)) * ($t | 1)) & 0xFFFFFFFF;
        $t ^= ($t + (($t ^ ($t >> 7)) * ($t | 61))) & 0xFFFFFFFF;
        $t = ($t ^ ($t >> 14)) & 0xFFFFFFFF;
        return $min + ($t % ($max - $min + 1));
    }
}
```

**Why Mulberry32:** small, well-distributed, deterministic across PHP versions. Instance-scoped — no global `mt_srand` state. Returns full 32-bit values which `BoardGenerator` already drives via `randFn(0, $max)` calls.

**`SeededRandom::rand` is called as `[$rng, 'rand']`** — drops directly into `BoardGenerator`'s existing `randFn` slot, no wrapper needed.

## Seed lifecycle

In `Game::setupNewGame` (around the existing [BoardGenerator construction](../../modules/php/Game.php)):

```php
$seed = (int)bga_rand(0, 2147483647);   // 31-bit positive int, fits SQL INT
$rng = new \SeededRandom($seed);
$generator = new \BoardGenerator(['randFn' => [$rng, 'rand']]);
$result = $generator->generate();

if (!$result['valid']) {
    throw new \BgaSystemException('Board generation failed');
}

$this->setGameStateValue('board_seed_decimal', $seed);
$this->setGameStateValue('board_algorithm_version', \BoardGenerator::ALGORITHM_VERSION);
$this->statInc($seed, 'board_seed_decimal');
$this->statInc(\BoardGenerator::ALGORITHM_VERSION, 'board_algorithm_version');
```

**Seed range:** `bga_rand(0, 2147483647)` gives a 31-bit positive value. Comfortably fits a SQL INT. A 31-bit search space (~2.1B) is collision-resistant for any plausible scale.

**Generation failure handling:** unchanged. The existing `maxBuildAttempts = 50` retry loop inside `generate()` consumes the seeded stream deterministically — a given seed either always succeeds or always fails. With the landscape bias on, empirical testing shows 100% success across hundreds of trials, so failure is a tail event. If it fires, the existing `throw` propagates and BGA retries table creation, which gets a fresh seed.

**Algorithm version:** `BoardGenerator::ALGORITHM_VERSION = 1`. Bump when the packing algorithm changes meaningfully (e.g., tuning `TARGET_ASPECT_RATIO`, changing scoring math, modifying placement rules). Old seeds remain recorded but cannot be regenerated against newer code; the regeneration script refuses version mismatches.

## Globals registration

In `Game::__construct` ([Game.php:38](../../modules/php/Game.php)):

```php
$this->initGameStateLabels([
    'board_seed_decimal' => 20,
    'board_algorithm_version' => 21,
]);
```

IDs 20 and 21 are within BGA's permitted 10–99 label range, intentionally chosen distinct from the stat IDs (11, 12) below to avoid eyeball confusion between the two namespaces. Future labels start at 22. Per-table, persistent for the table's lifetime.

**Backward compat:** Tables created before this feature shipped have `getGameStateValue` returning `0` (BGA's default for unset labels). Old games will report `(seed=0, version=0)`, which the regeneration script and any future UI must treat as "(seed unavailable)" rather than literal seed 0.

## Stats integration

Add to `stats.json` under `table`:

```json
"board_seed_decimal": {
    "id": 11,
    "name": "Board seed",
    "type": "int"
},
"board_algorithm_version": {
    "id": 12,
    "name": "Board algorithm version",
    "type": "int"
}
```

Stat IDs 11 and 12 are next-free (10 is `rounds_played`). **Once shipped, these IDs are immutable** — changing them later breaks all historical stats.

`statInc` calls in `setupNewGame` set the values. BGA's stats panel renders them automatically as e.g. `Board seed: 1234567890`. Players see the decimal form; the encoded `v1-...-...` form is for the developer regeneration script and any future UI.

## Encoding

`modules/php/BoardSeed.php`:

```php
class BoardSeed {
    private const ALPHABET = '0123456789ABCDEFGHJKMNPQRSTVWXYZ';  // Crockford base32

    public static function encode(int $seed, int $version): string {
        $seed = $seed & 0xFFFFFFFF;
        $chars = '';
        for ($i = 0; $i < 7; $i++) {
            $chars .= self::ALPHABET[$seed & 0x1F];
            $seed >>= 5;
        }
        return "v{$version}-" . substr($chars, 0, 4) . '-' . substr($chars, 4);
    }

    public static function decode(string $encoded): ?array {
        if (!preg_match('/^v(\d+)-?([0-9A-Z]{4})-?([0-9A-Z]{3})$/i', $encoded, $m)) {
            return null;
        }
        $version = (int)$m[1];
        $body = strtoupper($m[2] . $m[3]);
        $seed = 0;
        for ($i = 6; $i >= 0; $i--) {
            $idx = strpos(self::ALPHABET, $body[$i]);
            if ($idx === false) return null;
            $seed = ($seed << 5) | $idx;
        }
        return ['seed' => $seed, 'version' => $version];
    }
}
```

**32-bit seed → 7 base-32 chars.** 7 × 5 = 35 bits, comfortable headroom over the 31-bit seed.

**Crockford alphabet** (no I, L, O, U) prevents transcription confusion.

**Format:** `v{version}-XXXX-XXX` (e.g., `v1-K7F3-9DR`). 11 characters total. Decoder is permissive on case and on the second dash.

## Regeneration script

`tests/regenerate_board.php`:

```php
<?php
require_once(__DIR__ . '/../modules/php/SeededRandom.php');
require_once(__DIR__ . '/../modules/php/BoardSeed.php');
require_once(__DIR__ . '/../modules/php/BoardGenerator.php');

if ($argc < 2) {
    echo "Usage: php tests/regenerate_board.php <seed>\n";
    echo "  <seed> = decimal int or encoded form like 'v1-K7F3-9DR'\n";
    exit(1);
}

$input = $argv[1];
if (preg_match('/^v\d+/i', $input)) {
    $parsed = BoardSeed::decode($input);
    if ($parsed === null) { echo "ERROR: invalid encoded seed: $input\n"; exit(2); }
    $seed = $parsed['seed'];
    $version = $parsed['version'];
} else {
    $seed = (int)$input;
    $version = BoardGenerator::ALGORITHM_VERSION;
}

if ($version !== BoardGenerator::ALGORITHM_VERSION) {
    echo "ERROR: seed is for algorithm v{$version}; current is v"
       . BoardGenerator::ALGORITHM_VERSION . ".\n";
    echo "       Old seeds cannot be reproduced against the current algorithm.\n";
    exit(3);
}

$rng = new SeededRandom($seed);
$generator = new BoardGenerator(['randFn' => [$rng, 'rand']]);
$result = $generator->generate();

if (!$result['valid']) {
    echo "ERROR: generation failed for seed $seed after {$result['attempts']} attempts.\n";
    exit(4);
}

echo "Seed: $seed\n";
echo "Encoded: " . BoardSeed::encode($seed, $version) . "\n";
echo "Algorithm version: $version\n";
echo "Hexes: " . count($result['hexes']) . "\n";
echo "Cluster placements:\n";
foreach ($result['clusters'] as $i => $p) {
    printf("  [%2d] %-14s anchor=(%2d,%2d) rot=%d\n",
        $i, $p['cluster']['id'], $p['anchorQ'], $p['anchorR'], $p['rotation']);
}
$z = $result['zeusPosition'];
echo "Zeus: ({$z['q']}, {$z['r']})\n";
```

Run as: `php tests/regenerate_board.php 1234567890` or `php tests/regenerate_board.php v1-K7F3-9DR`.

## Testing

**Unit tests** (in `tests/test_board_generator.php`, appending to existing test sections):

1. **`SeededRandom` determinism:** two `SeededRandom(424242)` instances produce identical sequences for the first 100 `rand(0, N)` calls.
2. **`BoardSeed::encode`/`decode` roundtrip:** for several seeds (0, 1, 12345, 2147483647), `decode(encode(seed, 1))` returns `['seed' => seed, 'version' => 1]`.
3. **`BoardSeed::decode` rejects malformed input:** empty string, missing `v` prefix, wrong char count, lowercase that decodes wrong, alphabet violations (using `I`, `L`, `O`, `U`).
4. **Seeded board determinism:** two `BoardGenerator` instances with the same seed produce byte-identical `hexes`, `clusters`, and `zeusPosition`.
5. **Seed sensitivity:** seeds 424242 and 424243 produce different boards (catches accidental seed-ignoring bugs).

**Visual smoke (manual, optional):** run `php tests/regenerate_board.php` with several seeds; eyeball that placements look reasonable and reproducible.

## Open questions

None. All design decisions resolved:
- **Capture mechanism:** versioned 32-bit seed (Option A2 from brainstorm).
- **Format:** `v1-XXXX-XXX` Crockford base32 (Option 1c).
- **Display:** stats panel only — no live UI (Option 2b reinterpreted as "stats only").
- **JS:** no JS reproduction (Option 2a — PHP-only).
- **Storage:** BGA globals via `initGameStateLabels` (Option 1b).
- **Encoding helper:** kept (Option A from late refinement) — small cost, used by regen script and any future UI.
